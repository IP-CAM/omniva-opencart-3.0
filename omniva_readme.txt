Instalation needs.

* ftp, or related software allowing to reach shop files (FileZilla)
* OmnivaLT plugin (simply comes in compressed zip folder)
* Opencart admin panel administrator permisions (install, edit page availability etc.)

Instalation guide 

* unzip OmnivaLt module plugin.
* Upload unziped files to your shop root directory.

    * Attention! Do not overide files if you dont know it is save.
    ** Attention if you already have vqmod installed, 
        upload just admin, catalog, system folders to / dir.
        and vqmod/xml files to corresponding your shop dir.
    ** If you dont have vqmod installed upload all provided files
        Make sure not overide files if you dont know it doesnt break e-shop.
        * After upload completes, provide write permisions to vqmod/cache folder,
            and checked.cache, mods.cashe files.
        * Once it's done go to yourshop.com/vqmod/install - this will install it - done!!!.
    * Now omnivalt module is available, go to system/Users/User Groups and apply permisions 
        to omnivalt module pages. Install and provide required info within Omnivalt settings.
    * Attention!!! Make test  order to make sure it's all working properly.
    * If where comes an errors relative vqmod or so, check vqmod is configured properly. 
      Delete Vqmod caches vqmod/cache, check its fixed.
        * If module installed together with vqmod, it has caches itself, hence whose files are loaded.
            In order to listen xml file changes, upload xml folder at last, 
            or add enter space to any xml file and save - caches will reload with your shop data.

* If any questions arose please contact by email with your issue.



Diegimo reikmenys . 

    * ftp, ar susieta programinė įranga, leidžianti pasiekti parduotuvės failus (FileZilla, etc.) 
    * OmnivaLT  įskiepis (zip ar kitoks failas) 
    * Opencart administratoriaus teises 
    (įdiegti, redaguoti puslapio prieinamumą ir taip toliau), 
    
Instaliavimo žingsniai 

    * Išarchyvuoti OmnivaLt modulio programą. 
    * Įkelti failus į parduotuvės ftp serverį. 
    * Dėmesio! Neperašinėkite failų, jei jūs nežinote, kad tai yra, saugu. 
    ** Dėmesio, jei jūs jau turite įdiegtą vqmod, įkelkite tik admin, catalog, system aplankus. 
        ir vqmod/xml rinkmenos į atitinkamą jūsų parduotuvės direktoriją. 
    ** Jei jūs neturite įdiegto vqmod, įkelkite visus failus. Neperašinėkite failų, jei jūs nežinote, kad tai yra, saugu. 
    * Suteikite write permisions vqmod/cache aplankui, 
        ir checked.cache, mods.cashe failams. 
    * Naršyklėje atverkite yourshop.com/vqmod/install - tai įdiegs vqmod į jūsų sistemą - baigta!!!. 
    * Dabar omnivalt modulis yra pasiekiamas, nueikite  Sistemos/Vartotojo/Vartotojo Grupes 
        ir pritaikykite permisions omnivalt modulio puslapiams. 
        Įdiekite ir aprūpinkite reikalingą informaciją Omnivalt nustatymų viduje. 
    * Dėmesio!!! Padarykite bandomąjį užsakymą įsitikinti, kad viskas dirba tinkamai. 
    * Jei, kur atsiranda klaidų susijusių su vqmod ar pan, patikrinkite, 
        kad vqmod yra sukonfiguruotas tinkamai. Pašalinkite Vqmod cache vqmod/cache aplanke, perkraukite puslapį. 
    *, Jei modulis įdiegtas kartu su vqmod, jis turi cache aplanke ir jos kraunamos. 
        Kad klausytumėte xml failų pasikeitimų, išsiųskite xml aplanką vėliausiai, 
        arba redaguokite bet kuri xml kataloge esantį xml faila pridedant space/enter ir išsaugokite.
        Tai informuoja vqmod, jog failai pasikeite ir cache bus automatiskai perkrauti su naujiausiais
        paruotuvės duomenimis.
        
    *, iškilus klausimams rašykite elektroniniu paštu su jūsų svarstoma problema.
